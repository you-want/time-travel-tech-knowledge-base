# 文档切片规则详细说明

## 目的

本文档定义了本知识库 Markdown 文档的标准切片规则，确保 AI 向量检索时能获得高质量、结构清晰的片段。

## 切片策略

### 一级切片：按二级标题（##）

每篇技术文档的结构都是统一的，按二级标题切分：

```markdown
# 文档标题                    ← 不单独切片，作为文档级元数据
## 基础元数据                  ← 不单独切片，提取为元数据
## 一、现代底层原理（简化）     ← 切片 1
## 二、古代实操完整步骤         ← 切片 2
## 三、优势对比               ← 切片 3
## 四、局限性与短板           ← 切片 4
## 五、拓展延伸配套技术         ← 切片 5
```

### 二级切片：按段落长度

如果某个二级标题下的内容超过 1200 字，按自然段再细分：

- 每个自然段为一个子切片
- 子切片的元数据继承父切片的分类、难度等信息

### 三级切片：列表项

表格中的每一行不作为独立切片（表格整体作为一个切片的一部分）。

## 元数据提取规则

### 从文档头部提取

每篇文档的开头都有标准元数据块，需要提取为切片的元数据：

```markdown
## 基础元数据
- 分类：农业
- 落地难度：当日落地
- 必备原料：秸秆、人畜粪便、杂草、泥土
- 是否需要特殊工具：否
- 适用场景：生存致富、增产
- 标签：#低成本手工 #无电力 #高收益 #增产
```

提取为：

```json
{
  "分类": "农业",
  "落地难度": "当日落地",
  "必备原料": "秸秆、人畜粪便、杂草、泥土",
  "是否需要特殊工具": false,
  "适用场景": ["生存致富", "增产"],
  "标签": ["#低成本手工", "#无电力", "#高收益", "#增产"],
  "文档标题": "堆肥与有机肥制作",
  "文档路径": "docs/agriculture/compost-fertilizer.md"
}
```

### 从正文提取

每个切片还需要提取：

- **字数**：用于判断是否需要合并
- **关键词**：从标题和内容中提取（可用 TF-IDF 或简单词频统计）
- **内部链接**：文档中的 `[](path.md)` 链接，记录为关联文档

## 特殊文档处理

### 避坑文档（limits/）

避坑文档的切片在检索时应标记为"负面过滤"。当用户问题的意图与避坑文档匹配时，系统应主动提醒。

### 导航文档（index.md、roadmap.md）

导航文档不宜作为主要检索结果，权重应设为 0.3。它们的主要作用是引导用户到其他技术文档。

### 多文档关联

当切片中包含内部链接（如 `[堆肥](../agriculture/compost-fertilizer.md)`）时：

1. 记录链接目标文档的路径
2. 在检索结果中，如果用户问题同时涉及链接的两篇文档，应同时展示

## 编码与格式

- 所有文档使用 UTF-8 编码
- 切片后保留原始 Markdown 格式（包括标题、列表、表格）
- 不转换特殊字符和 Emoji

## 验证规则

切片完成后，应验证：

1. 每篇文档都被完整切片（没有遗漏）
2. 每个切片的字数在 200-1200 范围内
3. 元数据完整提取（分类、难度、标签）
4. 内部链接被正确识别
5. 没有空切片
